بررسی دستور mkdir

از دستور mkdir در لینوکس برای ساخت پوشه توسط ترمینال استفاده می کنیم؛ mkdir مخفف make directory است.

فرم کلی:

mkdir "directory_name"

اگه بخوایم تو یک مسیر متفاوت از مسیری که در ترمینال هستیم پوشه ایجاد کنیم باید مسیر رو کامل نوشته و سپس نام پوشه ای که میخوایم بسازیم رو بنویسیم.

فرض کنید میخوایم یک پوشه به نام myDir‍1 در مسیر /home/username/myDir0 ایجاد کنیم.

mkdir "/home/username/myDir0/myDir1"

میتونیم از آرگومان p استفاده کنیم تا پوشه هایی که در مسیر وجود ندارن هم ایجاد بشن؛ p ابتدای کلمه ی parent است.

در مثال قبل فرض کنید پوشه ی myDir0 نیز وجود نداره.

mkdir -p "/home/username/myDir0/myDir1"

میتونیم هنگام ساخت پوشه سه حالت خوندن، نوشتن و اجرا شدن رو برای پوشه ای که ساخته میشه تعیین کنیم.

در زیر سطح دسترسی پوشه ای که میسازیم رو برای سه گروه user, group و other به read محدود کردیم.

mkdir -m "ugo=r" "/home/username/myDir0/myDir1"

خلاصه

  • از دستور mkdir در لینوکس برای ساخت پوشه ی جدید توسط ترمینال استفاده میکنیم؛ mkdir مخفف make directory است.
